# 秒杀系统网关层优化方案

> 基于 maiya-platform Spring Cloud Gateway 的限流、认证与秒杀令牌设计

## 架构位置

```
CDN → Nginx → Gateway（Spring Cloud Gateway） → 秒杀服务 → Redis Lua → MQ
                   │
        ┌──────────┼──────────┐
        ▼          ▼          ▼
   令牌桶限流   JWT 认证   秒杀Token 校验
```

## 一、现有网关问题分析

### 1.1 当前代码存在的问题

| 问题 | 代码位置 | 风险 |
|---|---|---|
| **JWT 未验签** | `JwtAuthGlobalFilter:59-68` | 只做了 Base64 解码提取 userId，任何伪造 Token 都能通过 |
| **无秒杀限流** | 未实现 | Sentinel 虽然引入了但没有秒杀专用限流规则 |
| **无秒杀 Token** | 未实现 | 抢购请求直接穿透到后端，无预发放机制 |
| **X-User-Id 来源不可信** | `JwtAuthGlobalFilter:46` | 下游服务信任这个 Header，网关需保证其真实性 |

## 二、令牌桶限流（Sentinel + 本地令牌桶）

### 2.1 双维度限流架构

```
网关限流分为两层，互相补充：

Sentinel（集群维度）     → 控制整条网关的总入口流量
```

```java
                .setCount(1000)
                .setBurst(500)

        // 加载到 Sentinel
        GatewayRuleManager.loadRules(rules);
    }
```

### 2.3 本地令牌桶（Redis 实现）

针对每个秒杀 SKU 的库存量，在 Redis 中维护令牌桶，网关同步消费：

```java
// FlashTokenBucket.java — 基于 Redis 的分布式令牌桶
@Component
public class FlashTokenBucket {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_TOKEN_BUCKET = "flash:token_bucket:";
    private static final String KEY_TOKEN_LAST  = "flash:token_last:";
    private static final String KEY_TOKEN_TS    = "flash:token_ts:";

    /**
     * 初始化令牌桶
     * @param fsId 秒杀活动 ID
     * @param maxTokens 最大令牌数（= 库存量）
     * @param rate 令牌填充速率（个/秒）
     */
    public void initBucket(String fsId, int maxTokens, int rate) {
        redisTemplate.opsForValue().set(KEY_TOKEN_BUCKET + fsId, String.valueOf(maxTokens));
        redisTemplate.opsForValue().set(KEY_TOKEN_LAST + fsId, String.valueOf(maxTokens));
        redisTemplate.opsForValue().set(KEY_TOKEN_TS + fsId, String.valueOf(System.currentTimeMillis()));
        // TTL = 活动持续时间 + 1 小时，活动结束后自动过期
    }

    /**
     * 尝试获取令牌（非阻塞）
     * @return true=获取成功，false=令牌不足
     */
    public boolean tryAcquire(String fsId) {
        // Lua 脚本：原子操作 — 按速率填充令牌 + 扣减
        String lua =
            "local key_bucket = KEYS[1] " +
            "local key_last   = KEYS[2] " +
            "local key_ts     = KEYS[3] " +
            "local max_tokens = tonumber(ARGV[1]) " +
            "local rate       = tonumber(ARGV[2]) " +
            "local now        = tonumber(ARGV[3]) " +

            // 读取当前状态
            "local tokens = tonumber(redis.call('get', key_bucket) or max_tokens) " +
            "local last_tokens = tonumber(redis.call('get', key_last) or max_tokens) " +
            "local last_ts = tonumber(redis.call('get', key_ts) or now) " +

            // 按时间间隔填充令牌
            "local delta = math.max(0, now - last_ts) " +
            "local filled = math.min(max_tokens, last_tokens + delta * rate / 1000) " +

            // 扣减
            "if filled >= 1 then " +
            "  filled = filled - 1 " +
            "  redis.call('set', key_bucket, filled) " +
            "  redis.call('set', key_last, filled) " +
            "  redis.call('set', key_ts, now) " +
            "  return 1 " +
            "else " +
            "  redis.call('set', key_bucket, filled) " +
            "  redis.call('set', key_ts, now) " +
            "  return 0 " +
            "end";

        List<String> keys = Arrays.asList(
            KEY_TOKEN_BUCKET + fsId,
            KEY_TOKEN_LAST   + fsId,
            KEY_TOKEN_TS     + fsId
        );

        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(lua, Long.class),
            keys,
            String.valueOf(fsInfo.getMaxTokens()),
            String.valueOf(fsInfo.getRate()),
            String.valueOf(System.currentTimeMillis())
        );

        return result != null && result == 1;
    }

    /**
     * 获取当前令牌数（用于监控）
     */
    public long getCurrentTokens(String fsId) {
        String val = redisTemplate.opsForValue().get(KEY_TOKEN_BUCKET + fsId);
        return val == null ? 0 : Long.parseLong(val);
    }
}
```

## 三、用户身份校验增强

### 3.1 改进 JWT 校验（增加验签）

```java
// FlashAuthGlobalFilter.java — 改造后的认证过滤器
@Component
@Order(-100)
public class FlashAuthGlobalFilter implements GlobalFilter {

    private static final Logger log = LoggerFactory.getLogger(FlashAuthGlobalFilter.class);

    private final AntPathMatcher pathMatcher = new AntPathMatcher();

    private static final String[] WHITE_LIST = {
        "/api/auth/**", "/api/goods/public/**",
        "/api/payment/notify/**", "/api/notify/public/**",
        "/actuator/**", "/swagger-ui/**", "/v3/api-docs/**",
        "/favicon.ico"
    };

    // 秒杀相关路径（需要额外校验）
    private static final String[] FLASH_PATHS = {
        "/api/flash/grab", "/api/flash/preheat"
    };

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        String path = request.getURI().getPath();

        // 白名单放行
        if (isWhiteList(path)) {
            return chain.filter(exchange);
        }

        // 提取 Token
        String authHeader = request.getHeaders().getFirst(HttpHeaders.AUTHORIZATION);
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            return unauthorized(exchange.getResponse(), "缺少认证Token");
        }
        String token = authHeader.substring(7).trim();

        try {
            // 完整 JWT 校验（签名 + 过期）
            Claims claims = jwtTokenProvider.validateAndParse(token);

            String userId = claims.getSubject();
            String userRole = claims.get("role", String.class);

            // 构建可信请求头（下游服务信任这些 Header）
            ServerHttpRequest mutatedRequest = request.mutate()
                    .header("X-User-Id", userId)
                    .header("X-User-Role", userRole != null ? userRole : "")
                    .header("X-Auth-Source", "gateway")
                    .header("X-Auth-Time", String.valueOf(System.currentTimeMillis()))
                    .build();

            return chain.filter(exchange.mutate().request(mutatedRequest).build());

        } catch (ExpiredJwtException e) {
            return unauthorized(exchange.getResponse(), "Token已过期");
        } catch (JwtException e) {
            return unauthorized(exchange.getResponse(), "Token无效");
        }
    }

    // ↓ 原有白名单和 unauthorized 方法保持不变 ...
}
```

### 3.2 JWT 校验核心（抽出独立组件）

```java
// JwtTokenProvider.java — JWT 校验组件
@Component
public class JwtTokenProvider {

    @Value("${jwt.secret}")
    private String secretKey;

    private SecretKey key;

    @PostConstruct
    public void init() {
        byte[] keyBytes = Decoders.BASE64.decode(secretKey);
        this.key = Keys.hmacShaKeyFor(keyBytes);
    }

    /**
     * 校验并解析 JWT
     * @throws JwtException 校验失败时抛出
     */
    public Claims validateAndParse(String token) {
        return Jwts.parser()
                .verifyWith(key)              // 验签
                .requireIssuer("maiya-auth")   // 校验签发者
                .clockSkewSeconds(30)          // 30 秒时钟偏差容忍
                .build()
                .parseSignedClaims(token)
                .getPayload();
    }
}
```

### 3.3 网关 Header 安全（防止伪造）

下游服务必须在网关层面保证 `X-User-Id` 等 Header 的**可信性**——入口处**剥离**用户可能伪造的 Header，再由网关注入真实值：

```java
// 在 FlashAuthGlobalFilter 中增加剥离逻辑
private ServerHttpRequest stripUntrustedHeaders(ServerHttpRequest request) {
    // 剥离客户端可能伪造的内部 Header
    return request.mutate()
            .headers(headers -> {
                headers.remove("X-User-Id");
                headers.remove("X-User-Role");
                headers.remove("X-Auth-Source");
                headers.remove("X-Internal-Validate");
            })
            .build();
}
```

## 四、秒杀令牌（Flash Token）设计

### 4.1 什么是秒杀 Token

秒杀 Token 是在秒杀活动开始前，客户端通过特定接口**预申请**的一个一次性令牌。抢购时必须携带此 Token 才能扣库存。

```
活动前（客户端预申请）:
  客户端 ──POST /api/flash/token?fsId=xxx──→ 网关校验身份 → 秒杀服务生成 Token → Redis 存储 → 返回 Token

活动时（客户端抢购）:
  客户端 ──POST /api/flash/grab?fsId=xxx──→ 网关校验 Token → 扣库存 → 异步下单
                        Header: X-Flash-Token: xxxxxx
```

### 4.2 Token 生成接口

```java
// FlashTokenController.java — 秒杀 Token 发放
@RestController
@RequestMapping("/api/flash")
public class FlashTokenController {

    @Autowired
    private FlashTokenService flashTokenService;

    /**
     * 预申请秒杀 Token
     * 客户端在秒杀开始前调用，获取抢购资格
     */
    @PostMapping("/token")
    public ApiResult<String> acquireToken(
            @RequestParam String fsId,
            @RequestHeader("X-User-Id") String userId) {

        // 1. 校验活动是否存在且即将开始
        // 2. 校验用户资格（等级、黑名单等）
        // 3. 限频（一个用户对一个活动只能申请 1 个 Token）
        String token = flashTokenService.generateToken(fsId, userId);

        if (token == null) {
            return ApiResult.error("Token 发放已达上限或您无资格");
        }
        return ApiResult.success(token);
    }

    /**
     * 查询用户是否已获取 Token
     */
    @GetMapping("/token/status")
    public ApiResult<Boolean> hasToken(
            @RequestParam String fsId,
            @RequestHeader("X-User-Id") String userId) {
        return ApiResult.success(flashTokenService.hasToken(fsId, userId));
    }
}
```

### 4.3 Token 生成与存储

```java
// FlashTokenService.java — 令牌生成与校验
@Service
public class FlashTokenService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_TOKEN_POOL   = "flash:token_pool:";    // Set：活动已发放的 Token
    private static final String KEY_TOKEN_USER   = "flash:token_user:";    // String：用户→Token 映射
    private static final String KEY_TOKEN_USED   = "flash:token_used:";   // Set：已使用的 Token
    private static final String KEY_USER_TOKENS  = "flash:user_tokens:";  // Set：用户已持有的 Token

    /** Token 有效期 = 活动开始后 5 分钟 */
    private static final long TOKEN_TTL_SECONDS = 300;

    /**
     * 生成 Token
     */
    public String generateToken(String fsId, String userId) {
        // 1. 检查该用户是否已持有 Token（一人一 Token）
        String userTokenKey = KEY_USER_TOKENS + userId + ":" + fsId;
        Boolean exists = redisTemplate.hasKey(userTokenKey);
        if (Boolean.TRUE.equals(exists)) {
            return null; // 已持有，不再发放
        }

        // 2. 检查 Token 池是否还有余量（= 库存量）
        String poolKey = KEY_TOKEN_POOL + fsId;
        Long remain = redisTemplate.opsForSet().size(poolKey);
        if (remain != null && remain <= 0) {
            return null; // Token 已发放完毕
        }

        // 3. 生成 Token（UUID + 签名，防止伪造）
        String rawToken = UUID.randomUUID().toString().replace("-", "");
        String signature = hmacSign(fsId, userId, rawToken);
        String flashToken = rawToken + "." + signature;

        // 4. 存储
        redisTemplate.opsForSet().add(KEY_TOKEN_POOL + fsId, flashToken);
        redisTemplate.opsForValue().set(KEY_TOKEN_USER + userId + ":" + fsId, flashToken, TOKEN_TTL_SECONDS, TimeUnit.SECONDS);
        redisTemplate.opsForSet().add(KEY_USER_TOKENS + userId + ":" + fsId, flashToken);
        redisTemplate.expire(KEY_TOKEN_POOL + fsId, TOKEN_TTL_SECONDS, TimeUnit.SECONDS);

        return flashToken;
    }

    /**
     * 校验并消费 Token（网关或秒杀服务调用）
     * @return 校验结果
     */
    public boolean consumeToken(String fsId, String userId, String flashToken) {
        // 1. 校验签名
        String[] parts = flashToken.split("\\.");
        if (parts.length != 2) return false;

        String expectedSign = hmacSign(fsId, userId, parts[0]);
        if (!expectedSign.equals(parts[1])) return false;

        // 2. 校验是否已使用（防止重放）
        Boolean used = redisTemplate.opsForSet().isMember(KEY_TOKEN_USED + fsId, flashToken);
        if (Boolean.TRUE.equals(used)) return false;

        // 3. 标记已使用（原子操作）
        Long added = redisTemplate.opsForSet().add(KEY_TOKEN_USED + fsId, flashToken);
        if (added == null || added == 0) return false; // 已被其他请求标记

        redisTemplate.expire(KEY_TOKEN_USED + fsId, TOKEN_TTL_SECONDS, TimeUnit.SECONDS);

        // 4. 删除用户映射
        redisTemplate.delete(KEY_TOKEN_USER + userId + ":" + fsId);

        return true;
    }

    /**
     * 用户是否持有 Token
     */
    public boolean hasToken(String fsId, String userId) {
        String userTokenKey = KEY_USER_TOKENS + userId + ":" + fsId;
        return Boolean.TRUE.equals(redisTemplate.hasKey(userTokenKey));
    }

    private String hmacSign(String fsId, String userId, String raw) {
        // 使用网关与秒杀服务共享的密钥签名
        String data = fsId + ":" + userId + ":" + raw;
        return DigestUtils.sha256Hex(data + SecretConstants.FLASH_TOKEN_SALT);
    }
}
```

### 4.4 Token 校验网关过滤器

```java
// FlashTokenCheckFilter.java — 网关层校验秒杀 Token
@Component
@Order(-50)  // 在认证过滤器之后执行
public class FlashTokenCheckFilter implements GlobalFilter {

    private static final Logger log = LoggerFactory.getLogger(FlashTokenCheckFilter.class);

    // 需要 Token 校验的路径
    private static final String[] TOKEN_REQUIRED_PATHS = {
        "/api/flash/grab"
    };

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_TOKEN_USED = "flash:token_used:";
    private static final String KEY_TOKEN_USER = "flash:token_user:";

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        String path = request.getURI().getPath();

        // 非秒杀抢购路径，跳过 Token 校验
        if (!requiresToken(path)) {
            return chain.filter(exchange);
        }

        // 检查请求方法（秒杀只允许 POST）
        if (!HttpMethod.POST.matches(request.getMethod().name())) {
            return forbidden(exchange.getResponse(), "请求方法不允许");
        }

        // 从 Header 获取秒杀 Token
        String flashToken = request.getHeaders().getFirst("X-Flash-Token");
        if (flashToken == null || flashToken.isEmpty()) {
            return forbidden(exchange.getResponse(), "缺少秒杀令牌");
        }

        // 获取用户 ID（由之前的认证过滤器注入，可信）
        String userId = request.getHeaders().getFirst("X-User-Id");
        if (userId == null || userId.isEmpty()) {
            return unauthorized(exchange.getResponse(), "未认证用户");
        }

        // 校验 Token
        String fsId = request.getQueryParams().getFirst("fsId");
        if (fsId == null) {
            // 从 body 读取（POST 请求）
            fsId = extractFsIdFromBody(request);
        }

        boolean valid = verifyToken(fsId, userId, flashToken);
        if (!valid) {
            return forbidden(exchange.getResponse(), "秒杀令牌无效或已使用");
        }

        // 标记 Token 已消费（放入请求属性，下游服务不再重复校验）
        exchange.getAttributes().put("flashTokenConsumed", true);
        exchange.getAttributes().put("flashFsId", fsId);

        return chain.filter(exchange);
    }

    /**
     * 校验 Token（仅校验是否存在和是否已使用，不做签名验证）
     * 签名验证在秒杀服务中再做一次
     */
    private boolean verifyToken(String fsId, String userId, String flashToken) {
        // 1. 检查是否已使用
        Boolean used = redisTemplate.opsForSet().isMember(KEY_TOKEN_USED + fsId, flashToken);
        if (Boolean.TRUE.equals(used)) return false;

        // 2. 检查是否发放给该用户
        String userKey = KEY_TOKEN_USER + userId + ":" + fsId;
        String stored = redisTemplate.opsForValue().get(userKey);
        return flashToken.equals(stored);
    }

    private boolean requiresToken(String path) {
        for (String p : TOKEN_REQUIRED_PATHS) {
            if (pathMatcher.match(p, path)) return true;
        }
        return false;
    }

    // 从 Body 提取 fsId（响应式读取）
    private String extractFsIdFromBody(ServerHttpRequest request) {
        // 略 — 需缓存 body 后读取，或要求客户端同时放在 URL 参数
        return request.getQueryParams().getFirst("fsId");
    }

    // forbidden / unauthorized 方法略 ...
}
```

### 4.5 Token 完整生命周期

```
┌────── 活动前 ──────┐    ┌────── 活动中 ──────┐    ┌──── 活动后 ────┐
                     │    │                     │    │               │
  客户端预申请 Token     │    携带 Token 抢购       │    Token 自动过期
  POST /api/flash/token │   POST /api/flash/grab  │    Redis TTL 清理
      ↓                 │        ↓                │        ↓
  生成 Token + 签名     │   网关校验 Token         │    Token Set 过期
  存入 Redis            │   → 扣库存               │    Redis 自动删除
  返回客户端            │   → Token 标记已使用      │
                     │    → 异步下单             │
                     │                         │
```

### 4.6 Token 设计要点

| 要点 | 说明 |
|---|---|
| **一人一 Token** | 每个用户对一个活动只能申请一次 Token，防止囤货 |
| **一次性使用** | 使用后立即标记已消费，防止重放攻击 |
| **签名防伪造** | Token 末尾带 HMAC 签名，防止客户端篡改 |
| **短 TTL** | 有效期 5 分钟，活动结束后自动过期，不占内存 |
| **网关层校验** | 不合法 Token 在网关直接拒绝，不穿透到秒杀服务 |
| **秒杀服务二次校验** | 网关校验快（只查 Redis），秒杀服务做签名校验兜底 |

## 五、网关过滤器链路

```
请求进入
    │
    ▼
Filter 1: StripUntrustedHeadersFilter   剥离客户端伪冒的内部 Header
    │
    ▼
Filter 2: FlashAuthGlobalFilter         JWT 验签 → 注入可信 X-User-Id
    │
    ▼
Filter 3: FlashTokenCheckFilter         校验秒杀 Token（抢购接口）
    │
    ▼
Filter 4: SentinelGatewayFilter         令牌桶限流（QPS + 用户级别）
    │
    ▼
    路由 → 转发到秒杀服务
```

### Key Resolver（区分用户）

```java
// FlashKeyResolver.java — 限流 Key 提取
@Component("flashKeyResolver")
public class FlashKeyResolver implements KeyResolver {
    @Override
    public Mono<String> resolve(ServerWebExchange exchange) {
        // 优先按用户
        String userId = exchange.getRequest().getHeaders().getFirst("X-User-Id");
        if (userId != null) return Mono.just("flash:" + userId);

        // 降级按 IP
        String ip = exchange.getRequest().getRemoteAddress().getAddress().getHostAddress();
        return Mono.just("flash:ip:" + ip);
    }
}
```

## 七、全链路对比

| 维度 | 优化前 | 优化后 |
|---|---|---|
| **JWT 校验** | 无验签，Base64 直接解码 | HMAC 签名校验 + 过期验证 |
| **Header 安全** | 下游信任所有 Header | 网关剥离伪造 Header + 注入可信标记 |
| **入口限流** | Sentinel 有框架无规则 | 5000 QPS 令牌桶 + 用户级别 1 QPS |
| **SKU 限流** | 无 | Redis 令牌桶 + 漏桶双模式 |
| **防刷** | 无 | 秒杀 Token 预发放 + 一次性消费 |
| **抢购请求到秒杀服务** | 100% 穿透 | 网关拦截 90%+ 无效请求 |
