# 秒杀系统核心业务层优化方案

> Redis Lua 原子扣库存 + MQ 异步下单的核心链路优化

## 一、现有实现问题分析

### 1.1 Lua 脚本问题

```lua
-- 当前脚本（flash_stock.lua）

-- 问题1：活动检查 Key 拼接错误
local active = redis.call('EXISTS', 'flash:active:' .. KEYS[1])
-- KEYS[1] = flash:stock:FS001
-- 实际检查的 Key = flash:active:flash:stock:FS001  ← 根本查不到！
-- 正确应为：redis.call('EXISTS', 'flash:active:FS001')
-- 修复：活动 Key 应单独传入 KEYS[3] 或从 ARGV 获取 fsId

-- 问题2：DECR 后再 INCR 回滚
local stock = redis.call('DECR', KEYS[1])
if stock < 0 then
    redis.call('INCR', KEYS[1])  -- 回滚
    return -1
end
-- 问题：高并发下 INCR 回滚后库存可能只剩 0，但 DECR 已经扣超了
-- 更好做法：GET → 比较 → DECR（单一原子路径）

-- 问题3：用户购买计数在扣库存之后才设置 TTL
redis.call('INCR', 'flash:buy:' .. KEYS[1] .. ':' .. ARGV[1])
redis.call('EXPIRE', 'flash:buy:' .. KEYS[1] .. ':' .. ARGV[1], 86400)
-- 应在最开始就确保所有 Key 都有 TTL
```

### 1.2 Java 服务问题

| 问题 | 代码位置 | 影响 |
|---|---|---|
| 预热库存硬编码 100 | `preheatStock:59` | 每个活动固定 100，无法灵活配置 |
| 无请求幂等 Key | `grabFlashSale:95-98` | MQ 可能重复投递，依赖消费者去重 |
| MQ 发送失败不处理 | `grabFlashSale:100-102` | 库存已扣但 MQ 没发，订单丢失 |
| 无库存阈值告警 | 无 | 库存即将售罄无法感知 |

### 1.3 MQ 消费者问题

| 问题 | 影响 |
|---|---|
| 异常只 log 不 throw | RocketMQ 认为消费成功，消息丢失 |
| 无重试机制 | DB 临时故障导致订单创建失败无法恢复 |
| 幂等校验有并发窗口 | `findPendingOrder` + `insert` 之间可能重复插入 |
| 未处理消费者线程配置 | 默认线程池可能不适合秒杀场景 |

## 二、Redis Lua 脚本优化

### 2.1 脚本设计

```lua
-- flash_stock_v2.lua
-- Redis 原子扣库存 + 限购校验（优化版）
--
-- KEYS[1] = flash:active:{fsId}       活动状态
-- KEYS[2] = flash:stock:{fsId}        库存
-- KEYS[3] = flash:buy:{fsId}:{uid}    用户已购数量
-- KEYS[4] = flash:users:{fsId}        已购买用户集合
--
-- ARGV[1] = uid                       用户 ID
-- ARGV[2] = limitQty                  限购数量
-- ARGV[3] = ttl                       活动 TTL（秒）
--
-- 返回值：
--   >=0  → 剩余库存（扣减成功）
--   -1   → 库存不足
--   -2   → 活动未开始或已结束
--   -3   → 超出限购数量
--   -4   → 重复购买（已在用户集合中）

-- 1. 活动状态校验
local active = redis.call('EXISTS', KEYS[1])
if active == 0 then
    return -2
end

-- 2. 用户去重（防止同一用户并发重复扣库存）
local isMember = redis.call('SISMEMBER', KEYS[4], ARGV[1])
if isMember == 1 then
    return -4
end

-- 3. 限购校验
local buyCount = redis.call('GET', KEYS[3])
if buyCount and tonumber(buyCount) >= tonumber(ARGV[2]) then
    return -3
end

-- 4. 原子扣库存（先读后扣，避免 DECR 负数回滚的竞态）
local stock = redis.call('GET', KEYS[2])
if not stock or tonumber(stock) <= 0 then
    return -1
end

-- 执行扣减
redis.call('DECR', KEYS[2])

-- 5. 记录用户购买
redis.call('INCR', KEYS[3])
redis.call('EXPIRE', KEYS[3], 86400)

redis.call('SADD', KEYS[4], ARGV[1])
redis.call('EXPIRE', KEYS[4], tonumber(ARGV[3]))

-- 6. 返回剩余库存
local remain = redis.call('GET', KEYS[2])
return tonumber(remain)
```

### 2.2 关键设计变更

| 变更 | 优化前 | 优化后 |
|---|---|---|
| **活动检查** | `flash:active:flash:stock:FS001`（错误） | `flash:active:FS001`（正确） |
| **用户去重时机** | 扣库存后才 SADD，无去重 | SISMEMBER 前置校验，防止同用户并发 |
| **扣库存方式** | DECR → 负数 → INCR 回滚 | GET 判断 → DECR，单一路径 |
| **Key 传入** | 固定 2 个 KEYS，拼接混乱 | 4 个 KEYS 分离，职责清晰 |
| **库存预热弹性** | 硬编码 100 | 从活动配置读取 |

### 2.3 Redis 内存模型

```
flash:stock:FS001         → "1000"        (String, TTL=活动时长)
flash:active:FS001         → "1"           (String, TTL=活动时长)
flash:buy:FS001:u10086     → "1"           (String, TTL=86400)
flash:users:FS001          → {"u10086",     (Set,   TTL=活动时长)
                               "u10087",
                               "u10088"}
flash:token_bucket:FS001   → "950"         (String, 令牌桶)
flash:token_used:FS001     → {"tok001",     (Set,    Token 消费记录)
                               "tok002"}
flash:info:FS001           → "{"fsId":"...}"(String, TTL=30s, 给 CDN 同步)
```

## 三、Java 服务层优化

### 3.1 核心流程重构

```java
// FlashSaleHighConcurrencyServiceImpl.java — 重构后

@Service
public class FlashSaleHighConcurrencyServiceImpl implements FlashSaleHighConcurrencyService {

    private static final Logger log = LoggerFactory.getLogger(FlashSaleHighConcurrencyServiceImpl.class);

    @Autowired
    private StringRedisTemplate redisTemplate;
    @Autowired
    private FlashSaleMapper flashSaleMapper;
    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    private DefaultRedisScript<Long> stockLuaScript;

    private static final String KEY_ACTIVE  = "flash:active:";
    private static final String KEY_STOCK   = "flash:stock:";
    private static final String KEY_BUY     = "flash:buy:";
    private static final String KEY_USERS   = "flash:users:";
    private static final String KEY_REQ     = "flash:request:";  // 请求幂等

    @PostConstruct
    public void init() {
        stockLuaScript = new DefaultRedisScript<>();
        stockLuaScript.setLocation(new ClassPathResource("flash_stock_v2.lua"));
        stockLuaScript.setResultType(Long.class);
    }

    @Override
    @Transactional
    public JSONObject grabFlashSale(String fsId, String uid, String requestId) {
        JSONObject result = new JSONObject();

        // ---- 1. 请求幂等（防止 MQ 重试导致重复扣库存） ----
        String reqKey = KEY_REQ + requestId;
        Boolean firstSeen = redisTemplate.opsForValue().setIfAbsent(reqKey, "1", 30, TimeUnit.SECONDS);
        if (Boolean.FALSE.equals(firstSeen)) {
            // 相同 requestId 已处理过，返回之前的结果
            result.put("code", 0);
            result.put("msg", "请求已处理");
            return result;
        }

        // ---- 2. 从本地缓存或 Redis 获取活动配置（减少 DB 查询） ----
        FlashSale sale = getFlashSaleFromCache(fsId);
        if (sale == null || !sale.getEnable()) {
            result.put("code", -3);
            result.put("msg", "活动不存在或已关闭");
            return result;
        }

        // ---- 3. 执行 Lua 扣库存 ----
        Long ttl = (sale.getEndTime().getTime() - System.currentTimeMillis()) / 1000;
        List<String> keys = Arrays.asList(
            KEY_ACTIVE + fsId,
            KEY_STOCK  + fsId,
            KEY_BUY    + fsId + ":" + uid,
            KEY_USERS  + fsId
        );

        Long stockRemain = redisTemplate.execute(
            stockLuaScript, keys,
            uid,
            String.valueOf(sale.getLimitQty()),
            String.valueOf(Math.max(ttl, 60))
        );

        // ---- 4. 结果处理 ----
        if (stockRemain == null) {
            result.put("code", -99);
            result.put("msg", "系统繁忙");
            return result;
        }

        // 失败场景
        if (stockRemain < 0) {
            String msg;
            switch (stockRemain.intValue()) {
                case -1: msg = "库存不足";         break;
                case -2: msg = "活动未开始或已结束"; break;
                case -3: msg = "超出限购数量";      break;
                case -4: msg = "您已抢购过该商品";   break;
                default: msg = "抢购失败";
            }
            result.put("code", stockRemain.intValue());
            result.put("msg", msg);
            return result;
        }

        // ---- 5. 成功 → 发送 MQ（同步发送，确保可靠） ----
        JSONObject orderMsg = new JSONObject();
        orderMsg.put("fsId", fsId);
        orderMsg.put("uid", uid);
        orderMsg.put("requestId", requestId);
        orderMsg.put("timestamp", System.currentTimeMillis());

        try {
            // 同步发送，等待 Broker 确认
            SendResult sendResult = rocketMQTemplate.syncSend("FLASH_ORDER",
                orderMsg.toJSONString(), 2000);

            if (sendResult.getSendStatus() != SendStatus.SEND_OK) {
                // MQ 发送失败 → 库存已扣但消息未到 → 走补偿
                log.error("MQ发送失败, 加入补偿队列: fsId={}, uid={}, requestId={}",
                    fsId, uid, requestId);
                compensationQueue.add(fsId, uid, requestId);
                // 或直接异步重试
                asyncRetrySendMq(fsId, uid, requestId, 3);
            }

            log.info("MQ发送成功: fsId={}, uid={}, requestId={}, msgId={}",
                fsId, uid, requestId, sendResult.getMsgId());

        } catch (RocketMQException e) {
            // MQ 异常 → 库存已扣 → 补偿
            log.error("MQ异常, 加入补偿: fsId={}, uid={}, error={}",
                fsId, uid, e.getMessage());
            compensationQueue.add(fsId, uid, requestId);
        }

        // ---- 6. 库存阈值告警 ----
        if (stockRemain <= 10) {
            log.warn("秒杀库存即将售罄: fsId={}, remain={}", fsId, stockRemain);
            // 可触发告警或自动扩容
        }

        result.put("code", 0);
        result.put("msg", "抢购成功，正在排队处理");
        result.put("stockRemain", stockRemain);
        result.put("requestId", requestId);
        return result;
    }

    /**
     * 从本地缓存或 Redis 获取活动信息（减少 DB 查询）
     */
    private FlashSale getFlashSaleFromCache(String fsId) {
        // 先从本地 caffeine 缓存查询
        // 或从 Redis flash:info:{fsId} 读取
        // 兜底查 DB
        return flashSaleMapper.findByFsId(fsId);
    }
}
```

### 3.2 预热优化（支持批量）

```java
/**
 * 库存预热 — 支持批量
 */
@Override
public void preheatStock(String fsId) {
    FlashSale sale = flashSaleMapper.findByFsId(fsId);
    if (sale == null || !sale.getEnable()) {
        throw new BizException("活动不可用");
    }

    long ttl = (sale.getEndTime().getTime() - System.currentTimeMillis()) / 1000;
    if (ttl <= 0) {
        throw new BizException("活动已结束");
    }

    // 使用事务确保预热原子性
    sessionCallback.execute(new SessionCallback<Object>() {
        @Override
        public Object execute(RedisOperations ops) throws DataAccessException {
            ops.multi();
            ops.opsForValue().set(KEY_ACTIVE + fsId, "1", Duration.ofSeconds(ttl));
            ops.opsForValue().set(KEY_STOCK + fsId, String.valueOf(sale.getTotalStock()),
                Duration.ofSeconds(ttl));
            ops.opsForValue().set("flash:limit:" + fsId, String.valueOf(sale.getLimitQty()),
                Duration.ofSeconds(ttl));
            // 清除预热前的残留数据
            ops.delete(KEY_USERS + fsId);
            return ops.exec();
        }
    });

    log.info("库存预热完成: fsId={}, stock={}, ttl={}s", fsId, sale.getTotalStock(), ttl);
}

/**
 * 批量预热（线程池并行）
 */
@Async("flashPreheatExecutor")
public void batchPreheat(List<String> fsIdList) {
    fsIdList.parallelStream().forEach(this::preheatStock);
}
```

### 3.3 库存回滚优化（结合 MQ 事务状态）

```java
/**
 * 库存回滚 — 改进版
 * 场景：订单超时未支付 / 用户取消 / MQ 投递失败
 */
@Override
@Transactional
public boolean rollbackStock(String fsId, String uid, String requestId) {
    // 1. 幂等：防止重复回滚
    String rollbackKey = "flash:rollback:" + requestId;
    Boolean done = redisTemplate.opsForValue().setIfAbsent(rollbackKey, "1", 10, TimeUnit.SECONDS);
    if (Boolean.FALSE.equals(done)) {
        return true;  // 已回滚过
    }

    // 2. 回滚库存（Lua 保证原子性）
    String lua =
        "local stock = redis.call('INCR', KEYS[1]) " +
        "redis.call('SREM', KEYS[2], ARGV[1]) " +
        "redis.call('DECR', KEYS[3]) " +           // 减少用户购买计数
        "return stock";

    List<String> keys = Arrays.asList(
        KEY_STOCK + fsId,
        KEY_USERS + fsId,
        KEY_BUY   + fsId + ":" + uid
    );

    Long newStock = redisTemplate.execute(
        new DefaultRedisScript<>(lua, Long.class), keys, uid);

    log.info("库存回滚成功: fsId={}, uid={}, newStock={}, requestId={}",
        fsId, uid, newStock, requestId);

    return true;
}
```

## 四、MQ 消息优化

### 4.1 消息可靠性架构

```
扣库存成功
    │
    ├──▶ 同步发送 MQ（主路径）
    │        │
    │        ├── SEND_OK       → 消费者处理
    │        ├── FLUSH_DISK_TIMEOUT → 磁盘刷写超时（消息实际已写入）
    │        └── 网络异常       → 补偿队列
    │
    ├──▶ 写入补偿表（DB 兜底）
    │        │
    │        └── 补偿任务扫描未完成的消息 → 重新投递
    │
    └──▶ 返回用户"排队中"
```

### 4.2 消费者优化

```java
/**
 * FlashOrderConsumer.java — 优化版
 * 顺序消费 + 幂等 + 重试
 */
@Component
@RocketMQMessageListener(
    topic = MqTopic.FLASH_ORDER,
    consumerGroup = "consumer-flash-order",
    consumeMode = ConsumeMode.ORDERLY,          // 顺序消费
    consumeThreadMax = 32,                       // 秒杀场景提高消费线程
    consumeThreadNumber = 16,
    maxReconsumeTimes = 3                        // 最多重试 3 次
)
public class FlashOrderConsumer implements RocketMQListener<MessageExt> {

    @Autowired
    private FlashOrderMapper flashOrderMapper;
    @Autowired
    private FlashSaleMapper flashSaleMapper;

    // 本地去重缓存（防止同一消息短时间内重复消费）
    private final Cache<String, Boolean> dedupCache = Caffeine.newBuilder()
            .expireAfterWrite(10, TimeUnit.SECONDS)
            .maximumSize(100000)
            .build();

    @Override
    @Transactional
    public ConsumeConcurrentlyStatus onMessage(MessageExt msg) {
        String body = new String(msg.getBody(), StandardCharsets.UTF_8);
        String msgId = msg.getMsgId();

        try {
            JSONObject data = JSON.parseObject(body);
            String fsId   = data.getString("fsId");
            String uid    = data.getString("uid");
            String reqId  = data.getString("requestId");

            // ---- 1. 消息级别幂等 ----
            if (dedupCache.getIfPresent(msgId) != null) {
                return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
            }
            dedupCache.put(msgId, true);

            // ---- 2. 请求级别幂等（基于 requestId） ----
            FlashOrder existing = flashOrderMapper.findByRequestId(reqId);
            if (existing != null) {
                return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
            }

            // ---- 3. 获取秒杀商品信息 ----
            FlashSale sale = flashSaleMapper.findByFsId(fsId);
            if (sale == null) {
                log.error("秒杀活动不存在: fsId={}", fsId);
                return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
            }

            // ---- 4. 创建订单 ----
            FlashOrder order = new FlashOrder();
            order.setOrderId(generateOrderId(fsId, uid));
            order.setRequestId(reqId);
            order.setFsId(fsId);
            order.setUid(uid);
            order.setQty(1);
            order.setFlashPrice(sale.getFlashPrice());
            order.setGoodsInfoId(sale.getGoodsInfoId());
            order.setOrderStatus(OrderStatus.WAIT_PAY.getCode());
            order.setCreateTime(System.currentTimeMillis());

            flashOrderMapper.insert(order);

            log.info("秒杀订单创建成功: orderId={}, fsId={}, uid={}, reqId={}",
                order.getOrderId(), fsId, uid, reqId);

            return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;

        } catch (Exception e) {
            log.error("秒杀订单创建失败, 准备重试: msgId={}, body={}, error={}",
                msgId, body, e.getMessage());

            // 重试次数已达上限，写入死信队列
            if (msg.getReconsumeTimes() >= 3) {
                log.error("消息重试耗尽, 写入死信: msgId={}", msgId);
                saveToDeadLetter(msgId, body, e.getMessage());
                return ConsumeConcurrentlyStatus.CONSUME_SUCCESS;
            }

            return ConsumeConcurrentlyStatus.RECONSUME_LATER;
        }
    }

    private String generateOrderId(String fsId, String uid) {
        // 订单号 = 时间戳(12) + 机器ID(2) + 用户ID后4位 + 随机(6)
        String ts = String.valueOf(System.currentTimeMillis());
        String uidSuffix = uid.length() > 4 ? uid.substring(uid.length() - 4) : uid;
        String rand = UUID.randomUUID().toString().replace("-", "").substring(0, 6);
        return "F" + ts + uidSuffix + rand;
    }
}
```

### 4.3 消息防丢补偿机制

```java
// FlashOrderCompensationTask.java — 补偿扫描
@Component
public class FlashOrderCompensationTask {

    /**
     * 每 10 秒扫描一次：检查已扣库存但未创建订单的 "悬挂" 请求
     *
     * 实现思路：
     * 1. Redis 中维护一个 "处理中" 集合 flash:pending:{fsId}
     * 2. 扣库存成功时 SADD，订单创建成功时 SREM
     * 3. 补偿任务扫描长时间未移除的成员，重新投递 MQ
     */
    @Scheduled(fixedRate = 10000)
    public void compensatePendingOrders() {
        // 扫描所有活跃活动的 pending 集合
        Set<String> fsIds = redisTemplate.keys("flash:pending:*");
        for (String fsId : fsIds) {
            Set<String> pendingUsers = redisTemplate.opsForSet().members(fsId);
            for (String uid : pendingUsers) {
                // 检查是否已有订单（查 DB）
                int count = flashOrderMapper.countPendingByUid(uid, extractFsId(fsId));
                if (count == 0) {
                    // 库存已扣但订单未创建 → 重新投递 MQ
                    rocketMQTemplate.syncSend("FLASH_ORDER",
                        buildRetryMsg(extractFsId(fsId), uid), 2000);
                } else {
                    // 订单已创建，从 pending 中移除
                    redisTemplate.opsForSet().remove(fsId, uid);
                }
            }
        }
    }
}
```

## 六、监控与告警

### 6.1 Redis 关键监控

| 指标 | 命令 | 告警条件 |
|---|---|---|
| 当前库存 | `GET flash:stock:{fsId}` | < 10% 总库存 |
| 已购买用户数 | `SCARD flash:users:{fsId}` | 监控增速 |
| 令牌桶水位 | `GET flash:token_bucket:{fsId}` | < 20% 最大令牌 |
| 限流触发量 | Sentinel 监控 | 429 比例 > 30% |

### 6.2 MQ 关键监控

| 指标 | 说明 | 告警条件 |
|---|---|---|
| Consumer 堆积数 | RocketMQ 控制台 | > 10000 |
| 消费延迟 | 最后消费时间 - 当前时间 | > 30 秒 |
| 重试次数 | `getReconsumeTimes` | > 3 次 |
| 死信队列数量 | 死信 Topic 消息数 | > 100 |

### 6.3 业务监控

```java
// 秒杀核心指标上报（Prometheus / Micrometer）
@Aspect
@Component
public class FlashSaleMonitorAspect {

    private final Counter grabTotal;
    private final Counter grabSuccess;
    private final Counter grabFail;
    private final Gauge   stockRemain;

    public FlashSaleMonitorAspect(MeterRegistry registry) {
        grabTotal   = registry.counter("flash.grab.total");
        grabSuccess = registry.counter("flash.grab.success");
        grabFail    = registry.counter("flash.grab.fail");
        stockRemain = Gauge.builder("flash.stock.remain", this, FlashSaleMonitorAspect::getStock)
                       .register(registry);
    }

    @Around("execution(* ny.shop.youxuan.marketingservice..grabFlashSale(..))")
    public Object monitor(ProceedingJoinPoint pjp) throws Throwable {
        grabTotal.increment();
        Object result = pjp.proceed();
        JSONObject json = (JSONObject) result;
        if (json.getIntValue("code") == 0) {
            grabSuccess.increment();
        } else {
            grabFail.increment();
        }
        return result;
    }
}
```

## 八、优化前后对比

| 维度 | 优化前 | 优化后 |
|---|---|---|
| **Lua 活动检查** | Key 拼接错误，永远查不到 | 独立 KEYS 传入，正确校验 |
| **库存扣减** | DECR → 负数 → INCR 回滚（有竞态） | GET 判断 → DECR（单一原子路径） |
| **用户去重** | 无前置去重 | SISMEMBER 前置校验 |
| **幂等性** | 无 requestId | 请求级 + 消息级双重幂等 |
| **MQ 发送** | 异步 fire-and-forget | 同步发送 + 失败补偿 |
| **消费重试** | 异常只 log，消息丢失 | 3 次重试 → 死信队列 |
| **超时回滚** | 非原子操作 | Lua 原子回滚 + 幂等 |
| **监控** | 无 | Prometheus 指标 + 库存告警 |
