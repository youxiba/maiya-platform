## 流量收敛与写入效果

```
100万 并发请求
    │
    ▼
┌────────────────────────┐
│   CDN 边缘层             │  拦截 ≈ 90%
│   到源站 ≈ 10万          │
└──────────┬─────────────┘
           ▼
┌────────────────────────┐
│   Nginx 反向代理         │  拦截 ≈ 95%
│   到网关 ≈ 5千           │
└──────────┬─────────────┘
           ▼
┌────────────────────────┐
│   Spring Cloud Gateway  │  拦截 ≈ 90%
│   到秒杀服务 ≈ 500       │
└──────────┬─────────────┘
           ▼
┌────────────────────────┐
│  秒杀服务（核心业务）     │  Lua 原子扣库存 → MQ 削峰
│   请求幂等 + 补偿        │  500 QPS → 平滑至 2000 TPS 落库
└──────────┬─────────────┘
           ▼
┌────────────────────────┐
│  RocketMQ 异步落库       │  批量缓冲 100条 / 500ms
│  批量写入 ≈ 8000 TPS    │  延迟队列削峰 → DB 0 抖动
└────────────────────────┘
```

**全链路累计收敛：99.95%+** 的请求在到达秒杀服务前被拦截。
**DB 写入吞吐提升 6.6 倍**（1200 → 8000 TPS），写入曲线平滑无毛刺。
