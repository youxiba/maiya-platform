# 秒杀系统兜底与降级方案

> DB 压力过大时自动熔断，快速返回"排队中"

## 一、全链路降级层次

```
正常流量                   轻度压力                  重度压力                   灾难
────────────────────────────────────────────────────────────────────────────
CDN 缓存                   CDN 缓存                  CDN 缓存                 CDN 缓存
Nginx 限流                 Nginx 限流                Nginx 限流                Nginx 限流
Gateway 限流               Gateway 限流              Gateway 限流              Gateway 限流
                            │                          │                       │
秒杀服务正常                Redis Lua 降级              Redis Lua 降级            全部降级
MQ 消费正常                MQ 消费正常                 MQ 消费降速               MQ 停止消费
DB 正常                    DB 偶尔超时                 DB 连接池打满              DB 不可用
                            ↓                          ↓                        ↓
请求全部处理               熔断器打开，部分快速失败       全部快速返回排队中          返回"活动火爆"
```

## 二、熔断器实现

### 2.1 滑动窗口熔断器

```java
/**
 * FlashCircuitBreaker.java — 秒杀专用熔断器
 *
 * 基于滑动窗口统计 DB 调用失败率
 * 失败率 > 阈值 → 熔断打开 → 快速失败 → 自动恢复
 */
@Component
public class FlashCircuitBreaker {

    private static final Logger log = LoggerFactory.getLogger(FlashCircuitBreaker.class);

    // -------- 熔断器状态 --------
    public enum State {
        CLOSED,      // 正常：请求放行
        OPEN,        // 熔断：快速失败
        HALF_OPEN    // 半开：放行试探请求
    }

    // -------- 配置（从配置中心动态刷新） --------
    private volatile CircuitBreakerConfig config;

    // 滑动窗口（存储最近 N 个请求的结果）
    private final SlidingWindow window;

    // 当前状态
    private final AtomicReference<State> state = new AtomicReference<>(State.CLOSED);

    // 熔断打开时间戳（用于自动恢复）
    private final AtomicLong openedAt = new AtomicLong(0);

    // 半开状态下已放行的试探请求数
    private final AtomicInteger halfOpenRequests = new AtomicInteger(0);

    // 半开试探成功计数
    private final AtomicInteger halfOpenSuccesses = new AtomicInteger(0);

    // -------- 指标（Prometheus） --------
    private final Counter totalRequests;
    private final Counter failedRequests;
    private final Counter circuitBreakCount;

    public FlashCircuitBreaker(MeterRegistry registry) {
        this.config = new CircuitBreakerConfig();
        this.window = new SlidingWindow(config.getWindowSize());

        this.totalRequests = registry.counter("flash.cb.total");
        this.failedRequests = registry.counter("flash.cb.fail");
        this.circuitBreakCount = registry.counter("flash.cb.open");
    }

    /**
     * 判断请求是否允许通过
     * @return true=放行, false=熔断
     */
    public boolean tryAcquire() {
        totalRequests.increment();

        State currentState = state.get();

        switch (currentState) {
            case CLOSED:
                return true;

            case OPEN:
                // 检查是否到达自动恢复时间
                if (System.currentTimeMillis() - openedAt.get() >= config.getRecoveryTimeoutMs()) {
                    // 尝试切换到 HALF_OPEN
                    if (state.compareAndSet(State.OPEN, State.HALF_OPEN)) {
                        halfOpenRequests.set(0);
                        halfOpenSuccesses.set(0);
                        log.info("熔断器转入 HALF_OPEN, 开始试探恢复");
                        return true;
                    }
                }
                circuitBreakCount.increment();
                return false;

            case HALF_OPEN:
                // 半开状态：限制试探请求数量
                if (halfOpenRequests.incrementAndGet() <= config.getHalfOpenMaxRequests()) {
                    return true;
                }
                circuitBreakCount.increment();
                return false;

            default:
                return true;
        }
    }

    /**
     * 记录成功
     */
    public void onSuccess() {
        State currentState = state.get();

        if (currentState == State.HALF_OPEN) {
            int successes = halfOpenSuccesses.incrementAndGet();
            // 连续成功达到阈值 → 关闭熔断器
            if (successes >= config.getHalfOpenSuccessThreshold()) {
                if (state.compareAndSet(State.HALF_OPEN, State.CLOSED)) {
                    window.reset();
                    log.info("熔断器恢复 CLOSED, 服务恢复正常");
                }
            }
        }

        if (currentState == State.CLOSED) {
            window.record(true);
        }
    }

    /**
     * 记录失败
     */
    public void onFailure() {
        failedRequests.increment();

        State currentState = state.get();

        switch (currentState) {
            case CLOSED:
                window.record(false);
                // 检查失败率是否超过阈值
                double failureRate = window.getFailureRate();
                if (failureRate >= config.getFailureRateThreshold()) {
                    if (state.compareAndSet(State.CLOSED, State.OPEN)) {
                        openedAt.set(System.currentTimeMillis());
                        log.warn("熔断器打开! 失败率={}%, 窗口={}, 阈值={}%",
                            failureRate * 100, config.getWindowSize(),
                            config.getFailureRateThreshold() * 100);
                    }
                }
                break;

            case HALF_OPEN:
                // 试探请求失败 → 立即回到 OPEN
                if (state.compareAndSet(State.HALF_OPEN, State.OPEN)) {
                    openedAt.set(System.currentTimeMillis());
                    log.warn("熔断试探失败, 回到 OPEN 状态");
                }
                break;

            case OPEN:
                // 已熔断，无需处理
                break;
        }
    }

    /**
     * 获取当前状态
     */
    public State getState() {
        return state.get();
    }

    // -------- 滑动窗口 --------
    static class SlidingWindow {
        private final boolean[] results;
        private int index = 0;
        private int count = 0;

        SlidingWindow(int size) {
            this.results = new boolean[size];
        }

        synchronized void record(boolean success) {
            results[index % results.length] = success;
            index++;
            if (count < results.length) count++;
        }

        synchronized double getFailureRate() {
            if (count == 0) return 0.0;
            int failures = 0;
            int start = Math.max(0, index - results.length);
            for (int i = start; i < index; i++) {
                if (!results[i % results.length]) failures++;
            }
            return (double) failures / count;
        }

        synchronized void reset() {
            Arrays.fill(results, true);
            index = 0;
            count = 0;
        }
    }

    // -------- 动态配置 --------
    @Data
    public static class CircuitBreakerConfig {
        /** 滑动窗口大小 */
        private int windowSize = 100;
        /** 失败率阈值（0.0~1.0） */
        private double failureRateThreshold = 0.2;          // 20%
        /** 熔断自动恢复时间（毫秒） */
        private long recoveryTimeoutMs = 30_000;             // 30 秒
        /** 半开状态最大试探请求数 */
        private int halfOpenMaxRequests = 5;
        /** 半开状态连续成功阈值 */
        private int halfOpenSuccessThreshold = 3;

        // 从配置中心动态刷新 @RefreshScope
    }
}
```

## 三、三层熔断设计

### 3.1 本地熔断器（内存级别）

```
部署在每个秒杀服务实例中，感知本机 DB 连接状况

优点：无额外依赖，反应快（毫秒级）
缺点：单机感知，可能误判（本机网络抖动导致熔断）
```

### 3.2 分布式熔断器（Redis 级别）

```
通过 Redis 统计集群维度的 DB 失败率

优点：全局准确，防止单机误判
缺点：多一次 Redis 查询，增加延迟
```

```java
/**
 * DistributedCircuitBreaker.java — Redis 分布式熔断
 */
@Component
public class DistributedCircuitBreaker {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String KEY_FAIL_PREFIX  = "flash:cb:fail:";
    private static final String KEY_TOTAL_PREFIX = "flash:cb:total:";
    private static final String KEY_STATE        = "flash:cb:state";

    /**
     * 记录结果到 Redis（每分钟一个窗口）
     */
    public void record(String serverId, boolean success) {
        String minuteKey = getMinuteKey();
        String failKey  = KEY_FAIL_PREFIX  + minuteKey;
        String totalKey = KEY_TOTAL_PREFIX + minuteKey;

        redisTemplate.opsForValue().increment(totalKey);
        if (!success) {
            redisTemplate.opsForValue().increment(failKey);
        }
        redisTemplate.expire(failKey, 120, TimeUnit.SECONDS);
        redisTemplate.expire(totalKey, 120, TimeUnit.SECONDS);
    }

    /**
     * 检查集群维度的熔断状态
     */
    public boolean isCircuitOpen() {
        String state = redisTemplate.opsForValue().get(KEY_STATE);
        return "OPEN".equals(state);
    }

    /**
     * 计算最近 3 个窗口的失败率
     */
    public double getClusterFailureRate() {
        long totalFail = 0, totalAll = 0;
        // 最近 3 分钟
        for (int i = 0; i < 3; i++) {
            String key = getMinuteKey(i);
            String failStr  = redisTemplate.opsForValue().get(KEY_FAIL_PREFIX + key);
            String totalStr = redisTemplate.opsForValue().get(KEY_TOTAL_PREFIX + key);
            totalFail += failStr  != null ? Long.parseLong(failStr) : 0;
            totalAll  += totalStr != null ? Long.parseLong(totalStr) : 0;
        }
        return totalAll > 0 ? (double) totalFail / totalAll : 0.0;
    }

    private String getMinuteKey() { return getMinuteKey(0); }
    private String getMinuteKey(int offset) {
        long epochMinute = (System.currentTimeMillis() / 60000) - offset;
        return String.valueOf(epochMinute);
    }
}
```

### 3.3 异步熔断（MQ 消费延迟触发）

```java
/**
 * MqCircuitBreaker.java — 基于 MQ 堆积数的熔断
 *
 * 当 RocketMQ 消费延迟超过阈值时，自动降低消费速度
 * 甚至暂停消费，让 DB 喘口气
 */
@Component
public class MqCircuitBreaker {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    // RocketMQ 管理命令
    private final DefaultMQAdminExt admin;

    // 当前消费速率限制（动态调整）
    private final AtomicInteger maxConsumeSpeed = new AtomicInteger(1000);

    // 熔断级别
    public enum Level {
        NONE,       // 正常
        SPEED_LIMIT, // 限速 50%
        PAUSE       // 暂停消费
    }

    private final AtomicReference<Level> currentLevel = new AtomicReference<>(Level.NONE);

    /**
     * 检查堆积情况并调整消费速度
     */
    @Scheduled(fixedRate = 5000)
    public void adjustConsumeSpeed() {
        try {
            // 查询 FLASH_ORDER Topic 的堆积情况
            long consumerLag = getConsumerLag("FLASH_ORDER", "consumer-flash-order");
            long dbWriteTps  = getDbWriteTps();     // 最近 10 秒平均写入 TPS
            long dbMaxTps    = getDbMaxTps();        // DB 能承受的最大 TPS

            if (consumerLag > 50000 && dbWriteTps >= dbMaxTps * 0.8) {
                // DB 即将打满 + 堆积严重 → 暂停消费
                if (currentLevel.compareAndSet(Level.NONE, Level.PAUSE)
                    || currentLevel.compareAndSet(Level.SPEED_LIMIT, Level.PAUSE)) {
                    log.warn("MQ 消费暂停: lag={}, dbTps={}/{}", consumerLag, dbWriteTps, dbMaxTps);
                    pauseConsumers();
                }
            } else if (dbWriteTps >= dbMaxTps * 0.6) {
                // DB 压力较大 → 限速
                if (currentLevel.compareAndSet(Level.NONE, Level.SPEED_LIMIT)) {
                    log.warn("MQ 消费限速: dbTps={}/{}", dbWriteTps, dbMaxTps);
                    speedLimitConsumers(0.5);
                }
            } else if (dbWriteTps <= dbMaxTps * 0.3) {
                // DB 已恢复 → 恢复正常
                if (currentLevel.compareAndSet(Level.PAUSE, Level.NONE)
                    || currentLevel.compareAndSet(Level.SPEED_LIMIT, Level.NONE)) {
                    log.info("MQ 消费恢复: dbTps={}/{}", dbWriteTps, dbMaxTps);
                    resumeConsumers();
                }
            }

        } catch (Exception e) {
            log.error("调整消费速度失败", e);
        }
    }

    // 通过 RocketMQ admin API 调整消费者的线程数或拉取间隔
    private void pauseConsumers() { /* update consumer config */ }
    private void speedLimitConsumers(double ratio) { /* reduce thread count */ }
    private void resumeConsumers() { /* restore config */ }
}
```

### 3.4 三层熔断协作

```
请求到达 → 本地熔断器(CLOSED?)
              │
              ├── OPEN → 快速返回"排队中"
              │
              ▼ 通过
          分布式熔断器(CLOSED?)
              │
              ├── OPEN → 快速返回"排队中"
              │
              ▼ 通过
          MQ 堆积熔断器(消费中?)
              │
              ├── PAUSED → 返回"排队中"
              │
              ▼ 消费正常
          DB 写入(正常)
```

## 四、秒杀 API 降级实现

### 4.1 秒杀抢购降级

```java
/**
 * FlashSaleController.java — 带降级的秒杀入口
 */
@RestController
@RequestMapping("/api/flash")
public class FlashSaleController {

    @Autowired
    private FlashSaleHighConcurrencyService flashSaleService;

    @Autowired
    private FlashCircuitBreaker circuitBreaker;

    @Autowired
    private DegradeConfig degradeConfig;

    @PostMapping("/grab")
    public ApiResult<JSONObject> grabFlashSale(
            @RequestParam String fsId,
            @RequestHeader("X-User-Id") String uid,
            HttpServletRequest request) {

        // ---- 阶段 1: 检查全局降级开关 ----
        if (degradeConfig.isGlobalDegrade()) {
            return ApiResult.success(buildQueuingResponse(fsId, uid));
        }

        // ---- 阶段 2: 检查本地熔断器 ----
        if (!circuitBreaker.tryAcquire()) {
            log.warn("熔断器打开, 快速失败: fsId={}, uid={}", fsId, uid);
            degradeCount.increment();
            return ApiResult.success(buildQueuingResponse(fsId, uid));
        }

        // ---- 阶段 3: 检查分布式熔断器 ----
        if (distributedCircuitBreaker.isCircuitOpen()) {
            return ApiResult.success(buildQueuingResponse(fsId, uid));
        }

        // ---- 阶段 4: 正常执行 ----
        try {
            String requestId = request.getHeader("X-Request-Id");
            JSONObject result = flashSaleService.grabFlashSale(fsId, uid, requestId);

            circuitBreaker.onSuccess();
            return ApiResult.success(result);

        } catch (DataAccessException e) {
            // DB 异常 → 记录熔断
            circuitBreaker.onFailure();
            distributedCircuitBreaker.record(getServerId(), false);
            log.error("DB异常, 触发熔断: fsId={}, uid={}", fsId, uid, e);
            return ApiResult.success(buildQueuingResponse(fsId, uid));

        } catch (Exception e) {
            circuitBreaker.onFailure();
            log.error("抢购异常: fsId={}, uid={}", fsId, uid, e);
            return ApiResult.error("系统繁忙");
        }
    }

    /**
     * 构建"排队中"响应（降级/熔断时返回）
     */
    private JSONObject buildQueuingResponse(String fsId, String uid) {
        JSONObject resp = new JSONObject();
        resp.put("code", 0);
        resp.put("msg", "抢购成功，正在排队处理");
        resp.put("queuing", true);
        resp.put("estimated", 3000);  // 预计等待 3 秒
        return resp;
    }
}
```

### 4.2 全局降级开关

```java
/**
 * DegradeConfig.java — 降级开关
 * 通过配置中心动态调整（Nacos Config）
 */
@Component
@RefreshScope
public class DegradeConfig {

    /** 全局降级开关（DB 维护期间手动开启） */
    @Value("${flash.degrade.global:false}")
    private boolean globalDegrade;

    /** 降级后返回的预计等待时间（毫秒） */
    @Value("${flash.degrade.estimated-wait:5000}")
    private long estimatedWaitMs;

    /** 降级后依然放行的白名单用户 */
    @Value("${flash.degrade.whitelist:}")
    private String whitelistUsers;

    /**
     * 判断当前是否应降级
     */
    public boolean shouldDegrade(String uid) {
        // 全局降级
        if (globalDegrade) return true;

        // 白名单用户不降级
        if (whitelistUsers != null && whitelistUsers.contains(uid)) return false;

        return false;
    }
}
```

## 五、MQ 消费降级

### 5.1 消费速率自动调整

```java
/**
 * AdaptiveConsumer.java — 自适应消费速率
 */
@Component
public class AdaptiveConsumer {

    private static final Logger log = LoggerFactory.getLogger(AdaptiveConsumer.class);

    @Autowired
    private MeterRegistry registry;

    // 当前最大消费 TPS（动态调整）
    private final AtomicInteger maxTps = new AtomicInteger(2000);

    // DB 健康指标
    private volatile double dbConnectionUsage  = 0.0;   // 连接池使用率
    private volatile double dbWriteLatencyAvg  = 0.0;   // 写入平均延迟

    @Scheduled(fixedRate = 3000)
    public void adapt() {
        // 从 Micrometer 获取 DB 连接池使用率
        dbConnectionUsage = getGaugeValue("druid.connections.active") /
                            getGaugeValue("druid.connections.max");

        // 从 Micrometer 获取写入延迟
        dbWriteLatencyAvg = getTimerAvg("flash.db.write.time");

        // 计算新的最大 TPS
        int newMaxTps;
        if (dbConnectionUsage > 0.8 || dbWriteLatencyAvg > 500) {
            // DB 压力大 → 限速到 500
            newMaxTps = 500;
        } else if (dbConnectionUsage > 0.6 || dbWriteLatencyAvg > 200) {
            // DB 压力中 → 限速到 1000
            newMaxTps = 1000;
        } else {
            // DB 正常 → 恢复到 2000
            newMaxTps = 2000;
        }

        int oldTps = maxTps.getAndSet(newMaxTps);
        if (oldTps != newMaxTps) {
            log.info("自适应限速调整: {} → {}, connUsage={}%, latency={}ms",
                oldTps, newMaxTps,
                (int)(dbConnectionUsage * 100),
                (int)dbWriteLatencyAvg);
        }
    }
}
```

### 5.2 消费端背压

```java
/**
 * FlashBatchConsumer.java — 带背压的批量消费
 *
 * 根据 DB 压力动态调整消费速度
 */
@Component
@RocketMQMessageListener(
    topic = MqTopic.FLASH_ORDER,
    consumerGroup = "consumer-flash-order",
    consumeMode = ConsumeMode.ORDERLY,
    consumeThreadMax = 64,
    consumeThreadNumber = 32,
    batchSize = 32
)
public class FlashBatchConsumer implements RocketMQListener<MessageExt> {

    @Autowired
    private BatchOrderService batchOrderService;

    @Autowired
    private AdaptiveConsumer adaptiveConsumer;

    @Autowired
    private FlashCircuitBreaker circuitBreaker;

    @Override
    public ConsumeOrderlyStatus onMessage(List<MessageExt> msgs) {
        // ---- 1. 检查熔断器 ----
        if (!circuitBreaker.tryAcquire()) {
            // 熔断中 → 暂停消费 10 秒
            log.warn("消费熔断, 暂停 10s");
            return ConsumeOrderlyStatus.SUSPEND_CURRENT_QUEUE_A_MOMENT;
        }

        // ---- 2. 批量写入 ----
        long start = System.currentTimeMillis();
        try {
            List<FlashOrderMessage> messages = parseMessages(msgs);
            FlashOrderWriteResult result = batchOrderService.batchCreateOrders(messages);

            long cost = System.currentTimeMillis() - start;
            int tps = (int)(messages.size() / (cost / 1000.0 + 0.001));

            // ---- 3. 自动限速 ----
            int maxTps = adaptiveConsumer.getMaxTps();
            if (tps > maxTps) {
                // 超过限速阈值 → 慢一点
                long sleepMs = Math.min(1000, (tps - maxTps) * 10);
                Thread.sleep(sleepMs);
                log.debug("消费限速: tps={}, max={}, sleep={}ms", tps, maxTps, sleepMs);
            }

            circuitBreaker.onSuccess();
            return ConsumeOrderlyStatus.SUCCESS;

        } catch (Exception e) {
            circuitBreaker.onFailure();
            log.error("批量消费失败", e);
            return ConsumeOrderlyStatus.SUSPEND_CURRENT_QUEUE_A_MOMENT;
        }
    }
}
```

## 六、降级策略矩阵

### 6.1 按压力级别降级

```
级别0: 正常
  所有服务正常运行
  DB TPS < 2000, 连接池使用率 < 40%

级别1: 轻度压力
  DB TPS 2000~4000, 连接池使用率 40%~60%
  → MQ 消费限速 50%
  → 秒杀 API 正常

级别2: 中度压力
  DB TPS 4000~6000, 连接池使用率 60%~80%
  → MQ 消费限速 30%
  → 秒杀 API 本地熔断打开（快速返回排队中）
  → 活动信息接口降级到 Redis 缓存

级别3: 重度压力
  DB TPS > 6000, 连接池使用率 > 80%
  → MQ 消费暂停
  → 秒杀 API 全局降级（返回活动火爆）  
  → 分布式熔断器打开
  → 非核心服务全部关闭

级别4: 灾难
  DB 不可用
  → 秒杀 API 全部返回"活动火爆"
  → Redis 只读
  → 等待 DB 恢复后自动恢复
```

### 6.2 降级行为对照

| 用户操作 | 正常 | 轻度 | 中度 | 重度 | 灾难 |
|---|---|---|---|---|---|
| **抢购** | 实时处理 | 实时处理 | 排队中 | 活动火爆 | 活动火爆 |
| **查询活动** | DB | DB | 缓存(5s) | 缓存(30s) | 缓存(60s) |
| **查询订单** | DB | DB | 缓存(10s) | 缓存(60s) | 缓存 |
| **支付回调** | 同步 | 同步 | 同步 | MQ 异步 | MQ 异步 |
| **非核心接口** | 正常 | 正常 | 限流 | 降级 | 关闭 |

## 七、熔断恢复与健康检查

### 7.1 DB 健康检测

```java
/**
 * DbHealthChecker.java — DB 健康检测
 * 定期 ping DB，检测恢复后自动关闭熔断器
 */
@Component
public class DbHealthChecker {

    @Autowired
    private DataSource dataSource;

    @Autowired
    private FlashCircuitBreaker circuitBreaker;

    @Autowired
    private MqCircuitBreaker mqCircuitBreaker;

    /**
     * 每 5 秒检测 DB 健康状态
     */
    @Scheduled(fixedRate = 5000)
    public void checkHealth() {
        try (Connection conn = dataSource.getConnection()) {
            // 1. 检测连接是否可用
            boolean valid = conn.isValid(2);  // 2 秒超时

            // 2. 检测写入能力（执行轻量 write）
            if (valid) {
                try (Statement stmt = conn.createStatement()) {
                    stmt.execute("INSERT INTO health_check (ts) VALUES (NOW())");
                    stmt.execute("DELETE FROM health_check WHERE ts < NOW() - INTERVAL 1 MINUTE");
                }

                // 如果熔断器当前处于 OPEN 状态，尝试恢复
                if (circuitBreaker.getState() == FlashCircuitBreaker.State.OPEN) {
                    // 触发 HALF_OPEN
                    circuitBreaker.reset();
                    log.info("DB 健康检测通过, 尝试恢复熔断器");
                }

                // MQ 消费恢复
                mqCircuitBreaker.resume();
            }

        } catch (Exception e) {
            log.warn("DB 健康检测失败: {}", e.getMessage());
        }
    }
}
```

### 7.2 自动恢复流程

```
熔断器 OPEN (DB 压力大)
    │
    │ 等待 recoveryTimeoutMs (30 秒)
    ▼
熔断器 HALF_OPEN
    │
    │ 放行 5 个试探请求到 DB
    │
    ├── 全部成功 → CLOSED（恢复正常）
    │
    └── 任意失败 → OPEN（继续熔断）
         │
         │ 再次等待 30 秒
         ▼
       HALF_OPEN（循环直到成功）
```

## 八、兜底缓存策略

当 DB 不可用时，查询类接口从缓存兜底：

```java
/**
 * FlashQueryService.java — 多层缓存兜底
 *
 * 查询链路: DB → Redis → Local Cache → 默认值
 */
@Service
public class FlashQueryService {

    // 本地缓存（Caffeine，毫秒级响应）
    private final Cache<String, FlashSale> localCache;

    public FlashQueryService() {
        this.localCache = Caffeine.newBuilder()
                .maximumSize(1000)
                .expireAfterWrite(5, TimeUnit.SECONDS)
                .build();
    }

    /**
     * 查询活动信息（4 层兜底）
     */
    public FlashSale getFlashSale(String fsId) {
        // 第1层: 本地缓存
        FlashSale sale = localCache.getIfPresent(fsId);
        if (sale != null) return sale;

        // 第2层: Redis 缓存
        String json = redisTemplate.opsForValue().get("flash:info:" + fsId);
        if (json != null) {
            sale = JSON.parseObject(json, FlashSale.class);
            localCache.put(fsId, sale);
            return sale;
        }

        // 第3层: DB 查询
        try {
            sale = flashSaleMapper.findByFsId(fsId);
            if (sale != null) {
                // 回写 Redis
                redisTemplate.opsForValue().set("flash:info:" + fsId,
                    JSON.toJSONString(sale), 10, TimeUnit.SECONDS);
                localCache.put(fsId, sale);
                return sale;
            }
        } catch (DataAccessException e) {
            log.warn("DB 查询失败, 走默认兜底: fsId={}", fsId);
        }

        // 第4层: 默认值（内存固定数据，保证极端情况可用）
        return getDefaultFlashSale(fsId);
    }
}
```

## 九、降级大盘监控

### 9.1 熔断器状态

```java
// Actuator 端点 — 实时查看熔断器状态
@RestController
@RequestMapping("/internal/flash")
public class FlashDegradeController {

    @Autowired
    private FlashCircuitBreaker circuitBreaker;

    @Autowired
    private DistributedCircuitBreaker distributedCircuitBreaker;

    @Autowired
    private MqCircuitBreaker mqCircuitBreaker;

    @Autowired
    private DegradeConfig degradeConfig;

    /**
     * 降级状态大盘
     */
    @GetMapping("/degrade/status")
    public Map<String, Object> degradeStatus() {
        Map<String, Object> status = new LinkedHashMap<>();

        // 本地熔断器
        status.put("localCircuitBreaker", circuitBreaker.getState().name());
        status.put("localFailureRate", circuitBreaker.getFailureRate());

        // 分布式熔断器
        status.put("distributedCircuitBreaker",
            distributedCircuitBreaker.isCircuitOpen() ? "OPEN" : "CLOSED");
        status.put("clusterFailureRate", distributedCircuitBreaker.getClusterFailureRate());

        // MQ 消费状态
        status.put("mqConsumeLevel", mqCircuitBreaker.getCurrentLevel().name());

        // 全局降级
        status.put("globalDegrade", degradeConfig.isGlobalDegrade());

        return status;
    }

    /**
     * 手动降级（运维操作）
     */
    @PostMapping("/degrade/toggle")
    public String toggleDegrade(@RequestParam boolean enable) {
        degradeConfig.setGlobalDegrade(enable);
        log.warn("手动降级: {}", enable);
        return enable ? "降级已开启" : "降级已关闭";
    }
}
```

## 十一、降级响应体验

| 情景 | 用户看到 | HTTP 响应 |
|---|---|---|
| **正常抢购成功** | 抢购成功，正在排队处理 | `{"code":0,"msg":"排队中","queuing":false}` |
| **轻度降级** | 抢购成功，正在排队处理 | `{"code":0,"msg":"排队中","queuing":true,"estimated":3000}` |
| **中度降级** | 活动太火爆了，请稍后再试 | `{"code":1,"msg":"活动太火爆了","queuing":true}` |
| **重度降级** | 系统繁忙 | `{"code":-99,"msg":"系统繁忙"}` |
| **灾难降级** | 503 Service Unavailable | `{"code":503,"msg":"服务维护中"}` |

前端根据 `queuing=true` 展示倒计时或排队进度条，提升用户体验。
